# Creating the Lambda Layer
Here‚Äôs exactly how to zip your `requests` Lambda Layer using Windows **Command Prompt**.

---

### ‚úÖ Step-by-Step (Windows CMD) to Create `requests` Layer

#### üìÅ Folder structure you want:
```
requests-layer/
```

---

### ‚úÖ 1. Create the folder structure

Open Command Prompt and run:

```cmd
mkdir requests-layer
cd requests-layer
mkdir python
```

---

### ‚úÖ 2. Install `requests` into the `python` folder

Still inside `requests-layer`, run:

```cmd
pip install requests -t python
```

---

### ‚úÖ 3. Go one level up and zip it

Go back one level:

```cmd
cd ..
```

Then zip **only the `python/` folder**, so the ZIP file structure is valid for Lambda.

#### If you have PowerShell available (recommended):

```powershell
powershell Compress-Archive -Path requests-layer\python -DestinationPath requests-layer.zip
```

#### If PowerShell isn‚Äôt available and you have Git Bash or WSL, you can use `zip` there instead. Otherwise, install a tool like 7-Zip or use Windows File Explorer.

---

### ‚úÖ 4. Check ZIP contents (Important!)

Make sure `requests-layer.zip` contains this:

```
requests-layer.zip
‚îî‚îÄ‚îÄ python/
    ‚îî‚îÄ‚îÄ requests/
    ‚îî‚îÄ‚îÄ ...
```

> ‚ùó **If `python/` is *not* at the root** of the ZIP, Lambda will not recognize it as a valid layer.

---

### ‚úÖ 5. Upload to AWS Lambda Layer

Same steps:
- Go to AWS Console ‚Üí Lambda ‚Üí Layers
- Click **Create layer**
- Name it something like `requests-layer`
- Upload your `requests-layer.zip`
- Choose compatible runtimes (e.g. Python 3.9)
- Save

Then:
- Go to your Lambda function
- Scroll to **Layers**
- Click **Add a layer**
- Choose the one you just created
